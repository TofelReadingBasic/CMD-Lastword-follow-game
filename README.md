# 터미널 온라인 끝말잇기 (Windows cmd 기준)

혼자서 컴퓨터(AI)와 대결하거나, 여러 명이 같은 LAN에서 즐기는 끝말잇기 게임입니다.
단어는 국립국어원 **표준국어대사전 Open API**로 실시간 검증하며, 두음법칙(예: 량→양, 라→나, 녀→여)도 처리합니다.

정답을 맞히면 화면에 이렇게 표시됩니다.

```
[정답] 당신: 사과 | 명사 | 둥글고 단 과일.
```

AI 차례나 사전 확인 중에는 `AI가 단어를 고르는 중 /` 같은 회전 애니메이션이 나타납니다 (cmd에서도 정상 동작).

## 실행 방식 3가지

| 방식 | 파일 | 특징 |
|---|---|---|
| **혼자 하기 (vs AI)** | `singleplayer.py` | 접속 설정 없이 혼자서 컴퓨터와 대결. 난이도 선택 가능. |
| **완전 P2P** (여럿이) | `peer.py` | 중앙 서버 없음. 모두 동일한 스크립트를 실행하고 LAN에서 자동으로 서로를 찾아 직접 연결됩니다. |
| 중앙 서버 방식 | `server.py` + `client.py` | 한 사람이 서버를 켜 놓고, 나머지가 IP를 입력해 접속합니다. |

## 파일 구성
- `singleplayer.py` : 혼자 하기 버전 (컴퓨터 AI와 1:1 대결, 난이도 1~3 선택)
- `peer.py` : 완전 P2P 버전 (서버 없이 참가자 전원이 이 스크립트 하나만 실행)
- `server.py` / `client.py` : 중앙 서버 버전 (선택 사항)
- `hangul_utils.py` : 한글 분해/조합, 두음법칙, 끝말잇기 연결 규칙
- `dict_check.py` : 표준국어대사전 API 연동 단어 검증, 품사/뜻풀이/후보 단어 조회
- `ui_utils.py` : cmd에서 동작하는 로딩 스피너 애니메이션
- `requirements.txt` : 의존 패키지 목록

## 1. 준비 (Windows cmd)

명령 프롬프트(cmd)를 열고 파일이 있는 폴더로 이동합니다.

```bat
cd C:\Users\사용자이름\Downloads\files
pip install -r requirements.txt
```

`requirements.txt`가 없다면 이렇게 직접 설치해도 됩니다.

```bat
pip install requests
```

### 사전 API 키 발급 (필수)
1. https://stdict.korean.go.kr/openapi/openApiInfo.do 접속 후 무료 회원가입
2. "오픈 API 사용 신청" 탭에서 "저작권 정책 확인" 클릭
3. 사용 URL은 `http://localhost` 입력 후 "인증 키 발급" 클릭
4. 발급된 키를 cmd에 등록 (cmd 창을 새로 열 때마다 다시 입력해야 합니다)

```bat
set STDICT_API_KEY=발급받은_키_값
```

키가 없으면 모든 단어가 "사전 확인 불가"로 처리되어 게임을 진행할 수 없으니 꼭 등록하세요.

## 2. 혼자 하기 (vs AI) — 가장 간단하게 시작하는 방법

접속 설정이 전혀 필요 없습니다.

```bat
set STDICT_API_KEY=발급받은_키_값
python singleplayer.py
```

- 난이도를 고릅니다.
  - `1` 이지 : AI가 쉬운 단어(3글자 이하) 위주로 방어
  - `2` 하드 (기본값) : AI가 긴 단어(5글자 이하)까지 사용
  - `3` 익스트림 : AI가 글자 수 제한 없이 사전의 모든 단어를 동원 (매우 어려움)
- 먼저 시작할지(`y`) AI가 먼저 시작할지(`n`) 정합니다.
- 이후 서로 번갈아 단어를 냅니다. 정답을 맞히면 `단어 | 품사 | 뜻풀이` 형태로 표시됩니다.
- 오답을 내면 그 순간 게임이 끝나고, 대신 낼 수 있었던 단어 힌트 3개를 보여줍니다.
- `/quit` 을 입력하면 언제든 종료할 수 있습니다. 제한시간은 없습니다.

## 3-A. 완전 P2P로 실행하기 (여러 명, 별도 서버 불필요)

참가자 **전원**이 같은 와이파이(같은 공유기)에 연결된 상태에서, 각자 자신의 컴퓨터에서 실행합니다.

```bat
set STDICT_API_KEY=발급받은_키_값
python peer.py --name 철수
```

- 자동으로 같은 LAN에 있는 다른 참가자를 찾습니다. IP를 몰라도 됩니다.
- `/list` 를 입력하면 지금까지 발견된 참가자 목록이 보입니다.
- 참가자가 다 모이면 아무나 `/start` 를 입력합니다. 그 순간 발견된 인원 전체로 게임이 시작됩니다.
- 참가자들끼리 서로 직접 연결되어 진행되며, 중간에 멈추는 중앙 서버가 없습니다.
- 자신의 차례가 되면 단어를 입력하고, 아니면 입력한 내용은 채팅으로 전체에 전달됩니다.
- 게임이 끝나면 `/start` 를 다시 입력해 재시작할 수 있습니다.

옵션:
```bat
python peer.py --name 철수 --tcp-port 7001 --discovery-port 55201
```
- `--tcp-port` : 다른 참가자의 접속을 받을 포트 (기본은 자동 할당)
- `--discovery-port` : 탐색용 포트 (기본 55201). 참가자 전원이 같은 값을 써야 합니다.

> 참고: 각 참가자는 자신이 낸 단어에 대해서만 사전 검증을 직접 수행하고 그 결과를 다른 사람들에게 알리는 신뢰 기반 구조입니다.
> 회사/공용 와이파이처럼 자동 탐색(UDP 브로드캐스트)이 막혀 있는 환경에서는 3-B의 서버 방식을 이용하세요.

---

## 3-B. 중앙 서버로 실행하기 (선택 사항)

한 명이 서버 역할을 맡아 실행해 두고 나머지가 IP를 입력해 접속하는 방식입니다.

**서버 실행 (진행자 1명, cmd 창 하나 계속 열어둠)**
```bat
set STDICT_API_KEY=발급받은_키_값
python server.py 5555
```
- 같은 와이파이 안에서 함께 플레이한다면 서버 컴퓨터의 사설 IP(예: 192.168.0.5)를 참가자들에게 알려주세요. `ipconfig` 명령으로 IP를 확인할 수 있습니다.
- 인터넷을 통해 원격으로 모이려면 공유기 포트포워딩(5555/tcp)을 설정하거나 `ngrok tcp 5555` 같은 터널링 도구를 사용하세요.

**클라이언트 접속 (참가자 각자)**
```bat
python client.py 서버IP주소 5555
```
닉네임을 입력하면 대기실에 들어갑니다.

---

## 4. 게임 진행 (여러 명 모드 공통: P2P/서버)

- 참가자가 2명 이상 모이면 아무나 채팅창에 `/start` 를 입력해 게임을 시작합니다.
- 자신의 차례가 되면 안내 문구가 뜨고, 제한시간(기본 15초) 안에 단어를 입력하면 됩니다.
- 규칙:
  - 표준국어대사전에 등재된 2글자 이상 단어만 허용
  - 이전 단어의 마지막 글자로 시작해야 함 (두음법칙 자동 반영)
  - 이미 나온 단어는 재사용 불가
  - 제한시간 초과 또는 규칙 위반 시 탈락
  - 마지막까지 남은 한 명이 승리하며, 잠시 후 자동으로 대기실로 돌아가 재시작 가능
  - 정답을 맞히면 `단어 | 품사 | 뜻풀이` 형태로 화면에 표시됩니다.

## 5. 규칙/설정 조정
- `singleplayer.py` / `peer.py` / `server.py` 상단의 `TURN_TIME_LIMIT` 값을 바꾸면 제한시간을 조절할 수 있습니다.
- 최소 글자 수, 단어 재사용 허용 여부 등은 각 파일의 `_validate()` / `validate_word()` / `hangul_utils.is_valid_word_format()` 에서 수정할 수 있습니다.
- 로딩 애니메이션 문구나 속도는 `ui_utils.py`의 `Spinner` 클래스에서 바꿀 수 있습니다.

## 참고 (문제 해결)
- `python` 명령이 안 먹으면 `py` 로 시도해보세요 (`py singleplayer.py`).
- `pip install -r requirements.txt` 에서 파일을 못 찾으면 `pip install requests` 로 직접 설치해도 됩니다.
- 사전 API 요청이 많으면 응답이 느려질 수 있습니다. 같은 단어는 캐시되어 재검증하지 않습니다.
- 방화벽 때문에 접속이 안 되면 Windows 방화벽에서 해당 포트(및 P2P의 경우 UDP 55201)를 허용해 주세요.
- P2P 모드는 참가자가 각자 자신의 단어만 검증하는 신뢰 기반 구조입니다. 부정행위 방지가 필요하면 3-B의 중앙 서버 방식을 사용하세요.
